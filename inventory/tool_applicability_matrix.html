Two Netbox queries per run

The runner calls dcim/devices filtered by device_type for physical devices, then virtualization/virtual-machines for VMs. Both are scoped to tenant + site from the playbook.

W — Windows condition

AD and WSUS check the Netbox OS field before writing. If the device runs Linux, the runner skips the field update without error — it just logs skipped: os_family=linux.

Teramind — benchmark exclusion

Servers and VMs are permanently excluded. This is not a client override — it's in field_map.yaml as a hard rule. No client can override it here; they'd need a separate tool entry.

Nessus — ip_address fallback

Nessus returns IPs, not always hostnames. The runner matches by hostname first; if no match, it falls back to primary_ip in Netbox. Appliances often only match this way.

Client layer override

This matrix is benchmark defaults. Per-client exclusions (e.g. "no Teramind on VIP workstations") go in playbook.yaml under rules: — they never modify this file.

GLPI — universal fallback

Only tool covering all four object types. Use cf_glpi_last_seen as the existence check for appliances and any device type where no other tool has coverage.
